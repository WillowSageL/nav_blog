# 产品需求文档：赛博朋克风格 UI/UX 升级

**版本**: 1.0
**日期**: 2026-02-01
**作者**: Sarah (Product Owner)
**需求质量评分**: 92/100

---

## 执行摘要

本项目旨在将现有的 Next.js 14 + Supabase 个人导航页从当前的"AI 风味"极简商务设计，全面升级为**赛博朋克风格的二次元主题导航站**。通过引入动态粒子星空背景、3D 卡片悬浮效果、动漫风格插画图标以及交互式看板娘角色，打造具有高度视觉冲击力和社交分享价值的个性化产品。

**核心价值主张**：面向二次元爱好者社群，提供独特的赛博朋克美学体验，通过酷炫的视觉效果和有趣的交互设计，提升用户参与度和社交传播度，使产品在同类导航页中脱颖而出。

**预期影响**：
- 用户停留时间和互动次数显著增加
- 社交媒体分享率提升，获得至少 10+ 正面反馈
- 建立独特的品牌识别度，在二次元社群中形成口碑传播
- 为后续可能的开源或产品化奠定基础

---

## 问题陈述

**当前情况**：
现有导航页采用渐变背景（紫蓝青色系）+ 玻璃态效果的设计风格，虽然功能完善（书签管理、分类、搜索、主题切换），但视觉呈现过于"AI 产品化"——极简、商务、缺乏个性。这种设计风格在技术社区中已成为标配，难以在社交分享时引起关注和讨论。

**核心痛点**：
1. **视觉同质化**：与大量 AI 工具、SaaS 产品的界面风格雷同，缺乏辨识度
2. **情感连接弱**：冷静的商务风格无法与二次元爱好者的审美偏好产生共鸣
3. **分享价值低**：朋友看到后缺乏"哇"的惊喜感，不会主动询问或转发
4. **交互单调**：缺少有趣的动画和互动元素，用户体验偏向工具性而非娱乐性

**解决方案**：
通过赛博朋克风格的视觉重构和二次元元素的注入，将导航页从"工具"升级为"作品"——既保留原有的实用功能，又增加强烈的视觉吸引力和情感价值，使其成为用户愿意主动展示和分享的个性化产品。

**业务影响**：
- **短期**：提升用户参与度（停留时间 +50%）和社交传播度（3+ 人主动分享）
- **中期**：建立独特的品牌形象，在二次元社群中形成口碑
- **长期**：为可能的开源项目或商业化探索积累用户基础和技术经验

## 成功指标

### 主要 KPI

**1. 性能指标**（技术层面）
- **页面首次加载时间** < 3 秒（包含所有动画资源）
- **动画流畅度** ≥ 30fps（在主流设备上）
- **测量方法**：使用 Lighthouse 和 Chrome DevTools Performance 面板
- **验证时间**：Phase 1 完成后立即测试

**2. 用户认可度**（社交层面）
- **正面反馈数量** ≥ 10 条（评论、点赞、转发等）
- **来源渠道**：朋友圈、微博、小红书、技术社区等
- **测量方法**：手动统计社交媒体互动数据
- **验证时间**：Phase 1 上线后 2 周内

**3. 用户参与度**（行为层面）
- **平均停留时间**：相比当前基线增加 50%
- **互动次数**：点击看板娘、悬停卡片等交互行为增加
- **测量方法**：Google Analytics 或自建埋点
- **验证时间**：Phase 2 完成后持续监测

### 次要指标

- **社交传播度**：至少 3 位朋友主动询问或转发
- **移动端体验**：移动端加载时间 < 5 秒，简化版动画流畅运行
- **浏览器兼容性**：Chrome/Edge/Safari 最新两个版本完美支持

## 用户故事与验收标准

### Story 1：赛博朋克视觉风格

**作为** 二次元爱好者
**我想要** 看到深蓝、紫色、霓虹色调的赛博朋克配色方案
**以便** 界面能反映我的审美偏好，而不是千篇一律的商务风格

**验收标准**：
- [ ] 主色调采用深蓝（#0a0e27）、紫色（#6366f1）、霓虹粉（#ec4899）、霓虹青（#06b6d4）
- [ ] 背景使用深色基调，避免刺眼的亮色
- [ ] 文字和图标使用高对比度的霓虹色，确保可读性
- [ ] 支持暗色模式（默认）和亮色模式切换
- [ ] 移动端保持相同的配色方案

### Story 2：动态粒子星空背景

**作为** 用户
**我想要** 看到动态的粒子星空背景效果
**以便** 界面更有科幻感和沉浸感

**验收标准**：
- [ ] 背景包含缓慢移动的粒子效果（模拟星空）
- [ ] 粒子数量在 50-100 个之间，确保性能
- [ ] 粒子具有渐变透明度和微弱的发光效果
- [ ] 鼠标移动时粒子有轻微的交互反应（可选）
- [ ] 移动端简化为静态渐变背景或减少粒子数量
- [ ] 动画帧率保持在 30fps 以上

### Story 3：3D 卡片悬浮效果

**作为** 用户
**我想要** 书签卡片具有 3D 悬浮和倾斜效果
**以便** 交互更有趣，视觉更立体

**验收标准**：
- [ ] 鼠标悬停时卡片产生 3D 倾斜效果（跟随鼠标位置）
- [ ] 卡片具有悬浮阴影和发光边框
- [ ] 过渡动画流畅（使用 Framer Motion）
- [ ] 点击卡片时有缩放反馈动画
- [ ] 移动端简化为简单的缩放效果（无 3D 倾斜）
- [ ] 卡片内容（标题、图标）清晰可读

### Story 4：动漫风格图标和插画

**作为** 用户
**我想要** 看到手绘风格的图标和装饰性插画元素
**以便** 界面更有二次元氛围

**验收标准**：
- [ ] 分类图标采用线条风格或手绘风格
- [ ] 页面角落添加装饰性插画元素（如几何图形、线条装饰）
- [ ] 空状态页面使用插画而非纯文字
- [ ] 图标和插画风格统一，符合赛博朋克主题
- [ ] 所有图标支持 SVG 格式，确保清晰度

### Story 5：看板娘角色助手

**作为** 用户
**我想要** 一个可交互的看板娘角色
**以便** 界面更有趣，并能快速访问常用功能

**验收标准**：
- [ ] 看板娘固定在页面右下角或侧边
- [ ] 点击看板娘时显示动态表情和动作变化
- [ ] 看板娘提供快捷功能入口（搜索、添加书签、设置）
- [ ] 根据时间段显示不同的问候语或表情
- [ ] 看板娘可以最小化或隐藏（用户偏好设置）
- [ ] 移动端看板娘缩小尺寸或仅显示图标
- [ ] 使用 Live2D 或 Spine 动画（如技术可行）

## 功能需求

### 核心功能

**功能 1：赛博朋克配色系统**

**描述**：
实现完整的赛博朋克配色方案，包括深色基调、霓虹色强调、渐变效果和发光元素。

**用户流程**：
1. 用户打开页面，立即看到深蓝/紫色的背景
2. 卡片、按钮、文字使用霓虹色（粉、青、紫）
3. 交互元素具有发光效果（hover 时增强）
4. 用户可在设置中切换暗色/亮色模式

**边界情况**：
- 亮色模式下如何保持赛博朋克风格？→ 使用浅色背景 + 深色霓虹色
- 色盲用户如何识别？→ 确保足够的对比度，不仅依赖颜色区分

**错误处理**：
- CSS 变量加载失败 → 回退到默认配色
- 主题切换失败 → 保持当前主题，显示错误提示

**功能 2：动态粒子背景**

**描述**：
使用 Canvas 或 WebGL 实现动态粒子星空背景，粒子缓慢移动并具有发光效果。

**用户流程**：
1. 页面加载时初始化粒子系统
2. 粒子在背景层缓慢移动（速度可配置）
3. 鼠标移动时粒子产生轻微的交互反应（可选）
4. 用户可在设置中关闭粒子效果（性能优化）

**边界情况**：
- 低端设备性能不足 → 自动降级为静态渐变背景
- 用户禁用 JavaScript → 显示静态背景图片
- 浏览器不支持 Canvas → 优雅降级

**错误处理**：
- Canvas 初始化失败 → 回退到 CSS 渐变背景
- 动画帧率低于 20fps → 自动减少粒子数量或禁用效果

**功能 3：3D 卡片交互**

**描述**：
使用 Framer Motion 实现书签卡片的 3D 倾斜、悬浮阴影和点击反馈动画。

**用户流程**：
1. 用户鼠标悬停在卡片上
2. 卡片根据鼠标位置产生 3D 倾斜效果
3. 卡片显示悬浮阴影和发光边框
4. 用户点击卡片时产生缩放反馈
5. 卡片打开对应的书签链接

**边界情况**：
- 触摸设备无鼠标悬停 → 简化为点击时的缩放效果
- 卡片内容过长 → 使用省略号，hover 时显示完整内容
- 快速连续点击 → 防抖处理，避免重复触发

**错误处理**：
- Framer Motion 加载失败 → 回退到 CSS 过渡动画
- 3D 变换不支持 → 使用 2D 缩放效果

**功能 4：看板娘角色系统**

**描述**：
在页面右下角添加可交互的看板娘角色，提供动态表情、问候语和快捷功能入口。

**用户流程**：
1. 页面加载后看板娘出现在右下角（带入场动画）
2. 用户点击看板娘，显示对话框和功能菜单
3. 看板娘根据时间段显示不同的问候语
4. 用户可通过看板娘快速访问搜索、添加书签、设置等功能
5. 用户可在设置中最小化或隐藏看板娘

**边界情况**：
- 移动端屏幕小 → 缩小看板娘尺寸或仅显示图标
- 用户频繁点击 → 限制动画触发频率
- 看板娘遮挡内容 → 支持拖拽位置或自动避让

**错误处理**：
- 看板娘资源加载失败 → 显示占位图标，功能菜单仍可用
- 动画库加载失败 → 使用静态图片 + CSS 动画

### 不在范围内

以下功能**不包含**在本次升级中：
- 看板娘语音对话功能（仅文字交互）
- 复杂的 3D 场景渲染（仅卡片 3D 效果）
- 用户自定义配色方案（仅提供预设主题）
- 多语言支持（仅中文）
- 看板娘换装系统（Phase 2 后考虑）

## 技术约束

### 性能要求

**加载性能**：
- 首次内容绘制（FCP）< 1.5 秒
- 最大内容绘制（LCP）< 3 秒
- 首次输入延迟（FID）< 100ms
- 累积布局偏移（CLS）< 0.1

**运行时性能**：
- 动画帧率 ≥ 30fps（目标 60fps）
- 粒子系统 CPU 占用 < 10%
- 内存占用增加 < 50MB

**优化策略**：
- 使用 `requestAnimationFrame` 优化动画
- 粒子数量根据设备性能动态调整
- 图片和动画资源懒加载
- 使用 Web Workers 处理复杂计算（如需要）

### 安全要求

**数据安全**：
- 所有用户数据仍通过 Supabase RLS 保护
- 看板娘交互不涉及敏感数据传输
- 第三方动画库从可信 CDN 加载

**隐私保护**：
- 不收集用户交互行为数据（除非用户同意）
- 看板娘对话内容不上传服务器

### 集成要求

**现有系统集成**：
- 与现有的 Supabase 认证系统无缝集成
- 保持现有的书签管理、分类、搜索功能不变
- 兼容现有的主题切换系统（扩展为赛博朋克主题）

**第三方库**：
- **Framer Motion**：用于所有动画效果
- **React Spring**（可选）：用于粒子系统
- **Live2D SDK**（Phase 2）：用于看板娘动画

### 技术栈

**前端框架**：
- Next.js 14 (App Router)
- TypeScript
- React 18

**样式方案**：
- Tailwind CSS（扩展赛博朋克配色）
- CSS Variables（动态主题切换）
- Shadcn/ui（保留现有组件，增加自定义样式）

**动画库**：
- Framer Motion（主要动画库）
- Canvas API（粒子背景）

**兼容性**：
- **桌面端**：Chrome 90+, Edge 90+, Safari 14+, Firefox 88+
- **移动端**：iOS Safari 14+, Chrome Android 90+
- **降级策略**：不支持的浏览器显示静态版本

## MVP 范围与分阶段

### Phase 1：视觉风格重构（MVP 核心）

**目标**：建立赛博朋克视觉基础，实现核心视觉升级

**必须完成**：
1. **赛博朋克配色系统**
   - 深蓝/紫色/霓虹色配色方案
   - CSS 变量系统支持主题切换

2. **动态粒子星空背景**
   - Canvas 实现的粒子系统
   - 50-100 个粒子，缓慢移动
   - 性能优化和降级策略

3. **书签卡片重设计**
   - 赛博朋克风格的卡片样式
   - 霓虹色边框和发光效果
   - 悬停时的基础动画

**Phase 1 增强**（如时间允许）：
4. **3D 卡片悬浮效果**
   - Framer Motion 实现 3D 倾斜
   - 鼠标跟随效果
   - 点击反馈动画

5. **页面过渡动画**
   - 路由切换动画
   - 组件进入/退出动画
   - 加载状态动画

**交付标准**：
- 所有页面应用新视觉风格
- 性能指标达标（3秒加载 + 30fps）
- 桌面端和移动端（简化版）都可用
- 通过内部测试和朋友圈小范围分享

**预计工作量**：无时间限制，质量优先

**目标**：增加二次元元素和交互趣味性

**必须完成**：
1. **看板娘角色基础版**
   - 固定位置的看板娘图标
   - 点击显示对话框和功能菜单
   - 基础表情变化（2-3 种状态）
   - 快捷功能入口（搜索、添加、设置）

2. **动漫风格图标和插画**
   - 重绘分类图标为手绘风格
   - 添加装饰性插画元素
   - 空状态页面插画

**Phase 2 增强**（如时间允许）：
3. **看板娘高级功能**
   - Live2D 动画（如技术可行）
   - 更多表情和动作
   - 时间段问候语
   - 拖拽位置功能

4. **更多装饰元素**
   - 页面角落的几何装饰
   - 鼠标跟随光效
   - 滚动视差效果

**交付标准**：
- 看板娘功能完整可用
- 所有图标和插画风格统一
- 移动端体验优化

**预计工作量**：无时间限制，质量优先

| **兼容性风险**：旧浏览器不支持 Canvas/CSS 特性 | 低 | 低 | 1. 目标用户使用现代浏览器<br>2. 实现优雅降级，不支持的浏览器显示静态版本<br>3. 使用 feature detection 而非 browser detection |
